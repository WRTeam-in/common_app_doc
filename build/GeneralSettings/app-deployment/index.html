App Deployment

Overview​

This guide provides a comprehensive walkthrough for deploying your Flutter app to the Google Play Store. We'll cover everything from account setup to app submission, making it easy for non-technical users to follow.

Prerequisites​

Before starting the deployment process, ensure you have:

  1. A Google Play Developer Account ($25 one-time fee)
  2. Your Flutter app ready for release
  3. Basic understanding of using command line/terminal
  4. Access to your app's source code
  5. Required assets:
    • App icon (512x512 px)
    • Feature graphic (1024x500 px)
    • Screenshots for different devices (Phone, Tablet, TV)
    • Privacy policy URL
    • App description
    • Short description (up to 80 characters)
    • Full description (up to 4000 characters)
    • Promotional video (optional)

Step-by-Step Guide​

1. Developer Account Setup​

  1. Visit Google Play Console
  2. Choose your account type:
    • Individual developer (requires additional testing requirements)
    • Organization (requires D-U-N-S Number)
  3. Complete registration:
    • Pay $25 registration fee
    • Verify your identity
    • Set up payment profile
    • Provide contact information
    • Accept developer agreement
  4. For Organization accounts:
    • Obtain D-U-N-S Number (takes 1-30 business days)
    • Submit business verification documents
    • Provide organization details

2. App Configuration​

Before building your app, update these files:

Update pubspec.yaml​

version: 1.0.0+1  # Format: version_name+version_code

Configure Android Manifest​

Navigate to android/app/src/main/AndroidManifest.xml:

<manifest xmlns:android="http://schemas.android.com/apk/res/android">
    <application
        android:label="Your App Name"
        android:icon="@mipmap/ic_launcher">
        <!-- Your app configuration -->
    </application>
</manifest>

Update Gradle Settings​

In android/app/build.gradle:

android {
    defaultConfig {
        minSdkVersion 21
        targetSdkVersion 33
        versionCode flutterVersionCode.toInteger()
        versionName flutterVersionName
    }
}

3. Generate App Signing Key​

  1. Open terminal in project directory
  2. Run command:
keytool -genkey -v -keystore upload-keystore.jks -keyalg RSA -keysize 2048 -validity 10000 -alias upload
  1. Follow prompts to create keystore:
    • Enter password
    • Provide your details
    • Confirm information
  2. Store keystore file securely
  3. Make backup copies of the keystore

4. Configure Key Properties​

Create key.properties in android folder:

storePassword=<your keystore password>
keyPassword=<your key password>
keyAlias=upload
storeFile=<path to your keystore file>

Update android/app/build.gradle:

def keystoreProperties = new Properties()
def keystorePropertiesFile = rootProject.file('key.properties')
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(new FileInputStream(keystorePropertiesFile))
}

5. Build Your App​

  1. Open terminal in project directory
  2. Run command:
flutter build appbundle --release
  1. Optional flags:
    • --obfuscate - To protect your code
    • --split-debug-info - For crash reporting
  2. Wait for build completion
  3. Find app bundle at: build/app/outputs/bundle/release/app.aab

6. Play Store Setup​

Create App Listing​

  1. Go to Play Console
  2. Click "Create app"
  3. Fill basic information:
    • App name
    • Default language
    • App/game category
    • Free/paid status
    • Declaration of app content

Important Notes​

  • Keep keystore file safe for future updates
  • Ensure app complies with Play Store policies
  • Test thoroughly before submission
  • Keep version numbers updated
  • Monitor review status
  • Back up all signing keys
  • Document release process

Troubleshooting​

Common issues and solutions:

  1. Build Fails

    • Check Flutter version
    • Update dependencies
    • Verify Android SDK
    • Check Gradle configuration
    • Validate signing config

  2. Upload Issues

    • Verify keystore
    • Check version number
    • Validate assets
    • Confirm bundle size
    • Check API compatibility

  3. Review Rejection

    • Read rejection reasons
    • Make necessary changes
    • Update compliance
    • Improve documentation
    • Resubmit with updates

Best Practices​

  1. Test app thoroughly
  2. Keep documentation updated
  3. Maintain changelog
  4. Follow design guidelines
  5. Update content rating
  6. Monitor user feedback
  7. Plan regular updates
  8. Keep backups of all keys
  9. Document release process
  10. Stay compliant with policies

Need Help?​

If you encounter issues:

  1. Check Flutter Documentation
  2. Visit Play Console Help
  3. Consult Flutter community
  4. Review Play Store Policies
  5. Contact Play Store Support

Remember: First deployment may take longer. Subsequent updates will be faster.

Image Requirements​

For documentation maintainers, please ensure all screenshots meet these requirements:

  1. Resolution: Minimum 1280x720 pixels
  2. Format: PNG format with transparent background where applicable
  3. Content: Should be from the latest version of Google Play Console
  4. Text: All text should be clearly readable
  5. Highlighting: Key areas should be highlighted or annotated
  6. Personal Information: All sensitive information should be redacted
  7. Path: Save images in the /images/app-deployment/ directory
  8. Naming: Use kebab-case for image filenames (e.g., store-listing.png)
